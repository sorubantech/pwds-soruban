1 The Demo Cadence — how a module reaches management

Every business module travels the same weekly track. It is first shown to the team in the Friday internal demo; feedback is fixed over the weekend; then it is presented in the management demo on the following Monday.

2 Platform Features — what every screen shares

Cross-cutting capabilities that power all modules. Click a feature to expand its functionalities. A green ✓ means it is built & in the product today; hollow = on the roadmap.

A single configurable grid component reused on every list screen — never forked per screen.

  • Sorting — single & multi-column sort
  • Column control — show/hide, reorder, resize, per-user saved layout
  • Pagination — server-side paging & page-size selection
  • Row actions — inline edit / delete / toggle, driven by primary key config
  • Export — CSV / Excel of the current view
  • Alternate views — card grids & month calendars where it fits

A reusable filter builder that is EF-translatable and shared by grids and audience selection.

  • Conditions — multiple rules with field-type-aware operators
  • AND / OR groups — nested logic
  • Saved filters — reusable named filters (campaign library)
  • Grid bookmarks — per-user saved grid filters
  • Audience reuse — same builder powers email/campaign targeting + live counts

App-wide quick search across key entities.

  • Cross-entity search — contacts, donations, events & more
  • Jump-to-record — open the result directly

Each module has a dashboard built from configurable widgets.

  • Dashboard layouts — arrangeable per module
  • Widget types & properties — KPI cards, charts, lists
  • Role visibility — widget-by-role (Widget Role)
  • KPI cards — live metrics per business

One messaging engine across every channel, with typed merge tokens.

  • Channels — Email, SMS, WhatsApp, in-app Notification
  • Templates & campaigns — reusable templates, bulk campaigns, analytics
  • Placeholder definitions — typed merge tokens per entity
  • Per-tenant providers — company-level provider config (primary → fallback)

Getting external data in without creating duplicates.

  • Bulk import — CSV/Excel with field mapping
  • Duplicate detection & merge

Server-rendered public pages resolved per tenant by host, with their own branding.

  • Slug + host resolution — per-tenant routing, unique slug
  • Branding — colours, logo, OG/share image
  • Page types — donation, event reg, P2P/crowdfunding, prayer, volunteer, member portal
  • Lifecycle — Draft / Published / Active / Closed / Archived

A provider-pattern payment layer plus the money-handling around it.

  • Gateway providers — Braintree · Razorpay · PayU (pluggable)
  • Company gateways — per-tenant gateway + enabled methods
  • Pending → Confirmed — gateway callback confirms the payment
  • Receipts & tax — numbering, tax config, refunds, reconciliation
  • Multi-currency — currency master + conversion

Rules that fire actions on events in the system.

  • Triggers — on record/state changes
  • Actions — send comms, update data, queue jobs (Hangfire)

Templated document & certificate generation.

  • Certificate pipeline — template → process → print
  • Document types — managed document categories

The platform plumbing every module sits on.

  • Multi-tenant — every record company-scoped
  • Access control — users, roles, capabilities, role-capability, governance
  • Org hierarchy — company, branches, organizational units, staff
  • Master data & region — country → state → district → city → locality → pincode
  • Audit trail & integrations — change history, accounting/social/API/marketplace

The intelligence layer that surfaces insight and drafts content. Last in line — across both platform and business.

  • Engagement / generosity scoring
  • Churn prediction
  • Action board — next-best actions
  • AI draft — generate communication copy
  • Ask your data — natural-language reporting
  • Predictive analytics
