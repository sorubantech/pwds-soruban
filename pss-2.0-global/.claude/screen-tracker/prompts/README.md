# Screen Prompts

> This folder contains **rich screen development prompts** generated by `/plan-screens`.
> Each file is a complete prompt for one screen, including task tracking.
> `/build-screen` reads from this folder to execute screen generation.

## File Naming

`{entity-lower}.md` — e.g., `program.md`, `membership-tier.md`, `sms-template.md`

## File Format

Each prompt file has:

1. **Frontmatter** — metadata + status tracking
2. **Tasks** — checklist of generation pipeline steps
3. **Screen Prompt** — the rich prompt content fed to `/generate-screen`

## Statuses

| Status | Meaning |
|--------|---------|
| `PENDING` | Prompt generated, waiting for `/build-screen` execution |
| `IN_PROGRESS` | Currently being built (partially complete) |
| `PARTIALLY_COMPLETED` | Some tasks done (e.g., BE done, FE pending) |
| `COMPLETED` | All tasks done, screen fully generated |

## Workflow

```
/plan-screens → creates prompt file (status: PENDING)
/build-screen → reads prompt, executes pipeline, updates tasks
              → on partial: status = PARTIALLY_COMPLETED
              → on complete: status = COMPLETED + updates REGISTRY.md
```
